Que es Git

Git es un sistema de control de versiones distribuido que se utiliza para rastrear cambios en el código fuente durante el desarrollo de software. Aquí hay una descripción general básica de cómo funciona Git:

Repositorio

Git almacena la información en un repositorio. Puedes iniciar un nuevo repositorio o clonar uno existente.

Áreas de Trabajo

Git tiene tres áreas principales de trabajo: el directorio de trabajo (working directory), el área de preparación (staging area o index), y el repositorio local.

Flujo de trabajo básico

Cuando trabajas en un proyecto, haces cambios en los archivos en tu directorio de trabajo. Luego, seleccionas qué cambios deseas incluir en tu próximo commit al agregar esos cambios al área de preparación utilizando git add. Después de que hayas preparado tus cambios, puedes confirmarlos con un commit utilizando git commit. Esto guarda una instantánea de los archivos preparados en tu repositorio local.

Historial de Cambios

Git mantiene un historial de cambios, lo que permite retroceder a versiones anteriores si es necesario.

Ramificaciones (Branches)

Puedes crear ramas para trabajar en nuevas características o correcciones de errores sin afectar la rama principal (generalmente llamada master o main). Las ramas te permiten experimentar y fusionar tus cambios cuando estén listos.

Fusiones (Merges)

Después de que hayas completado el trabajo en una rama, puedes fusionar esos cambios de nuevo a la rama principal.

Repositorios remotos

Git es distribuido, lo que significa que puedes trabajar en un repositorio local y también en uno remoto. Los cambios se pueden enviar y recibir entre repositorios locales y remotos.

Conflictos

Puede haber conflictos cuando dos ramas tienen cambios en las mismas líneas de código. Git te alertará si esto sucede y te permitirá resolver esos conflictos manualmente.

Algunos comandos de Git comunes son:

  • git init: Inicia un nuevo repositorio.
  • git clone: Clona un repositorio existente.
  • git add: Agrega cambios al área de preparación.
  • git commit: Crea un nuevo commit con los cambios preparados.
  • git push: Envía cambios a un repositorio remoto.
  • git pull: Obtiene cambios del repositorio remoto.
  • git merge: Fusiona ramas.
